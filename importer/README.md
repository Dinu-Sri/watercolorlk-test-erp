# Google Reviews Importer

This folder contains all tools needed to import Google Business reviews into your Watercolor.LK store database.

## What It Does

1. **Reads** reviews from the JSON file generated by the scraper (`tmp-google-reviews-scraper-pro/google_reviews.json`)
2. **Downloads** user profile images and stores them locally in `reviews_images/profiles/`
3. **Stores** all review data in the `google_reviews` database table
4. **Enables** your product pages to display real customer reviews with avatars

## Prerequisites

Before running the importer:

1. **Run the scraper** to generate the JSON file:
   ```bash
   cd tmp-google-reviews-scraper-pro
   python start.py scrape --url "https://www.google.com/maps/place/Watercolor.LK/..." --max-reviews 100
   cd ..
   ```

2. **Ensure database tables exist** by running the schema:
   ```bash
   mysql -u root -p watercolorlk_store < schema.sql
   ```
   (Replace root/password with your actual credentials)

3. **Have PHP CLI installed** and accessible in your system PATH

## How to Run

### Option 1: Import (Keep Existing Reviews)

This adds new reviews and updates existing ones:

```bash
php importer/import.php
```

### Option 2: Fresh Import (Delete All & Re-import)

This clears all existing reviews and imports fresh from the JSON:

```bash
php importer/import.php --delete-existing
```

### Option 3: Custom JSON File

If your scraper JSON is in a different location:

```bash
php importer/import.php --json-file /path/to/custom_google_reviews.json
```

## What Happens During Import

```
✓ Validates JSON file exists
✓ Reads all reviews from JSON
✓ For each review:
  - Stores author, rating, text, date in database
  - Downloads user profile picture
  - Saves image locally with unique filename
  - Links image path to review record
✓ Generates import log with timestamp
✓ Displays summary statistics
```

## Database Schema

The `google_reviews` table stores:

```sql
- review_id (unique Google review identifier)
- place_id (Google business place ID)
- author (reviewer name)
- rating (1-5 stars)
- review_text (the review content)
- review_date (when review was posted)
- likes (review likes count)
- author_profile_url (link to reviewer's Google profile)
- profile_picture_local_path (path to downloaded avatar in reviews_images/)
- profile_picture_remote_url (original Google image URL)
- owner_response (business owner's response text)
- language (detected language code)
- is_active (1 = show on site, 0 = hidden)
- imported_at (when review was imported)
- updated_at (last update timestamp)
```

## Recurring Imports

To keep reviews fresh, you can:

1. **Schedule the scraper periodically** (daily/weekly via cron):
   ```bash
   # Add to crontab (runs scraper daily at 2 AM)
   0 2 * * * cd /path/to/watercolorlk && python tmp-google-reviews-scraper-pro/start.py scrape --url "..." -q
   ```

2. **Then run the importer**:
   ```bash
   # Add to crontab (runs import daily at 3 AM, after scraper completes)
   0 3 * * * cd /path/to/watercolorlk && php importer/import.php
   ```

3. **On your live server**: Pull changes and run manually or via webhook:
   ```bash
   git pull origin main
   php importer/import.php
   ```

## Troubleshooting

### JSON file not found error

Make sure you've run the scraper first in `tmp-google-reviews-scraper-pro/`:

```bash
cd tmp-google-reviews-scraper-pro
python start.py scrape --url "https://www.google.com/maps/place/Watercolor.LK/..." --max-reviews 50
cd ..
php importer/import.php
```

### Database connection errors

Verify your database credentials in `config/app.php` or `config/local.php`:

```php
define('DB_HOST', 'localhost');
define('DB_PORT', 3306);
define('DB_NAME', 'watercolorlk_store');
define('DB_USER', 'root');
define('DB_PASS', 'your_password');
```

### Image download failures

This is normal for a few images (e.g., if Google rate-limits or image is temporary). The importer will:
- Continue with other images
- Still store the review data
- Log which images failed

You can manually download failed images later or re-run the importer.

### Permission denied errors

On Linux/cPanel, ensure the `importer/reviews_images/` directory is writable:

```bash
chmod -R 755 importer/reviews_images/
```
